**Restaurant Database (DB Microsoft Access)**

Overview: This project is a Microsoft Access database designed to manage core restaurant operations, including orders, menu items, customers, supplies and staff. It demonstrates how relational database design can be used to organize and query business data efficiently.

The original dataset has been anonymized and replaced with sample data for portfolio and demonstration purposes.

Credits: Database structure was taught by Alex Araujo
Project Date: October 2025

--------------------------------------------------------------------------------------------------------------------
Features
Manage customer records
Track restaurant orders and order details
Organize menu items and pricing
Maintain staff information
Run SQL queries using Microsoft Access SQL syntax
Technology Used
Microsoft Access (.accdb)
Microsoft Access SQL
Database Structure (High-Level)

The database is structured around key relational tables such as:

Customers
Orders
Order Details
Menu Items
Staff

These tables are linked through primary and foreign key relationships to support efficient querying and reporting.

How to Use
Download the .accdb file from this repository
Open it using Microsoft Access
Explore tables, relationships, and queries in the navigation pane
Run or modify existing queries to analyze restaurant operations
Purpose of This Project

This project was built as part of a data portfolio to demonstrate:

Relational database design fundamentals
Practical use of Microsoft Access for business systems
Ability to structure and query operational data
